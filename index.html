Welcome to Rawr

What is Rawr?

Rawr is a word that means "I love you" in dinosaur, but also is this small library to implement a lot of features to the PHP language.

Rawr is an open-source project meant to make the PHP language a little better, as much as it has a lot of little failures that sometimes let me angry. The project started after a big criticism to PHP and its inconsistence in function names. To solve a big part of the problems, we got Rawr. It implements and simulates static-typing, functional programming, a more pure object-oriented approach and the concept of modules, aiming a more pure way to program and taking into account that so many developers do several mistakes and workarounds with PHP. Rawr is built to be type-safe and for applications that require applications to not crash.

Static typing

Different from Java where primitives aren't objects, here each type is literally a class and each value of a rawr-defined variable is pure object. As much as Alan Kays definition of object orientation, the objects have behaviours and Rawr also allows method-chaining to make the things easier. You can also restrict function parameters to specified types and existing comments over functions are parsed to assert type safety:

$age         = Int (18);
$ableToDrive = Lambda (
  /** :: (Ord a) => a -> Bool */
  function ($age) {
    return $age -> greaterOrEq (18);
  }
) -> invoke ($age);

It's also possible to use Rawr types as constraints to parameters:

$integralSum = Lambda (function (Int $n, Int $m) {
  return $m -> add ($n);
});

Functional programming

Rawr implements several functional concepts, as function-composition, and the rawr-defined functions have methods to work with reflection:

$multiplyTwo = Lambda (
  /** :: Int -> Int */
  function (Int $n) {
    return $n -> times (2);
  });

$multiplyThree = Lambda (
  /** :: Int -> Int */
  function (Int $n) {
    return $n -> times (3);
  });

# Function composition.
($multiplyTwo -> o ($multiplyThree)) -> invoke (Int (7)); # 42

More pure object orientation

In Rawr, almost all thing is an object. The behaviours of these objects is determined according to its type, e.g.: objects of Str class can have a side-effect behaviour called putStrLn to output its data to screen, as much as objects of bool atoms true and false can have behaviours like ifTrue or ifFalse:

use \Data\Num\Int;
$oddNumTil100 = (new Collection (1, 3, '...', 100)) -> of ('Data.Num.Int')
-> filter (function (Int $n) {
  return $n -> greaterThan (50); # Filters all the integers greater than 100.
})
-> each (function (Int $n) {
  $n -> show () -> putStrLn (); # Outputs to screen each one of them.
});

$itHappens = (Int (1) -> eq (Int (1)))
-> ifTrue (function () {
  Str ("Yes, they are the same.")
  -> putStrLn ();
})
-> ifFalse (function () {
  Str ("You newbie. They're different.")
  -> putStrLn ();
});
